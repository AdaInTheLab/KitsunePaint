# KitsunePaint DIY Kit

Run the same Unity asset-bundle compiler that powers
[paint.kitsuneden.net](https://paint.kitsuneden.net) on your own machine.
No web round-trip, no rate limits, no installation of Unity required.

This kit is the same `build_bundle.py` script our server runs ~ packaged
with its template bundle so you can compile paints locally for bulk work,
offline use, or integration into your own pipeline.

## Requirements

- Python 3.10 or newer
- Two pip packages:

  ```bash
  pip install UnityPy Pillow
  ```

## What's in the kit

```
KitsunePaint-DIY-Kit/
├── README.md                 # this file
├── build_bundle.py           # the compiler script
└── Atlas.template.unity3d    # the empty Unity bundle to inject textures into
```

## Quick start

1. **Make a Resources folder** containing your textures, named with the
   convention the script expects:

   ```
   my_paints/
       Resources/
           wood_oak_diffuse.png
           wood_oak_normal.png        # optional ~ generated default if missing
           wood_oak_specular.png      # optional ~ generated default if missing
           stone_smooth_diffuse.png
           stone_smooth_normal.png
           ...
   ```

   - `_diffuse.png` is required for each paint.
   - `_normal.png` and `_specular.png` are optional ~ defaults are generated
     if you skip them.
   - All textures are auto-resized to 512×512 if they're not already.

2. **Run the script** pointing at the Resources folder:

   ```bash
   python build_bundle.py path/to/my_paints/Resources
   ```

   You'll get one `Atlas_NNN.unity3d` bundle written into the same folder
   per paint name found.

3. **Use the bundles** however you like ~ as a 7DTD modlet, as raw Unity
   assets in another project, as input to your own pipeline.

## Pack ID namespacing

By default the script reads `../ModInfo.xml`'s `<Name>` field (or falls
back to the parent folder name) and prefixes every internal asset path
with that pack ID. This prevents collisions when multiple paint packs
load into the same Unity runtime.

Override with `--pack-id <id>` to set it explicitly, or pass an empty
string to disable namespacing entirely:

```bash
python build_bundle.py path/to/Resources --pack-id my_custom_id
python build_bundle.py path/to/Resources --pack-id ""        # no prefix
```

## Using the bundles outside 7 Days to Die

The `.unity3d` bundles contain three textures per paint
(`<packId>_<paint>_diffuse.png` + `_normal` + `_specular`) at 512×512 in
DXT1/DXT5 compression with 8 mip levels. They're standard Unity asset
bundles ~ any Unity-based game or tool with bundle import support can
consume them. The 7DTD modlet wrapper (painting.xml, ModInfo.xml,
Localization.txt) is generated by the [paint.kitsuneden.net](https://paint.kitsuneden.net)
web frontend and isn't part of this kit ~ if you want it, build via the
web tool instead.

## Troubleshooting

**"Mipmap warnings on game load"** ~ those come from OCBCustomTextures'
own built-in textures, not from KitsunePaint bundles. Cosmetic, every
paint mod produces them.

**"Atlas.template.unity3d not found"** ~ the template lives next to the
script. Run from inside the kit folder, or pass an absolute path to the
script.

**"Different texture sizes"** ~ the script forces 512×512 to match
vanilla. If your source is bigger or smaller, it's resized via Pillow's
LANCZOS filter. Pre-resize if you want pixel-perfect control.
